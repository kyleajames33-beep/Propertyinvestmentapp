# Referral touchpoints map

Every referral CTA across the 8 journey stages, by persona and professional type.

---

## How to read this document

Each stage section contains one or more touchpoints. Each touchpoint specifies:
- **Persona**: who sees this CTA
- **Professional**: who is being referred
- **Why now trigger**: what the user just learned that makes this referral relevant right now
- **Location**: which page and section the CTA appears in
- **CTA variants**: soft, medium, and direct copy options
- **Lead capture**: fields required for the professional to respond meaningfully

All CTAs follow the platform's referral disclosure (see `/disclosure`). No CTA appears before the user has consumed the educational content for that stage.

## Stage 4: Shortlisting

### Touchpoint 4.1: No primary referral CTA

All personas: Stage 4 (Shortlisting) has no referral CTA. This is a "do the work" stage where users apply the research framework to specific properties. The platform provides the comparison tool and checklist. A referral here would be premature and feel pushy.

**Exception:** If the user has not saved any properties after 10 minutes on the page, a gentle tooltip may appear suggesting they download the shortlisting checklist. No professional referral.

## Lead form design rules

1. **No form appears without context.** The user must scroll through the stage content before a CTA is displayed. Forms are contextually anchored to specific learning moments.

2. **Soft CTA is default.** The soft variant is the primary displayed text. Medium and direct variants are shown via A/B testing or progressive disclosure (user clicks "learn more" to see direct option).

3. **Maximum 9 fields per form.** If a form exceeds 9 fields, split into two steps: step 1 (contact details, 3 fields), step 2 (contextual details, remaining fields).

4. **Phone number is optional on all forms.** The user can submit with email only. Phone marked as "preferred contact method."

5. **File upload only on contract review forms.** No other stage asks for file uploads.

6. **Disclosure appears below every form.** Copy: "We receive a referral fee from professionals we introduce. This does not affect what you pay. You are under no obligation to proceed. Read our disclosure." (links to `/disclosure`)

7. **Confirmation state.** After submission, the user sees: (a) confirmation message, (b) estimated response time (24 to 48 hours), (c) "Continue your journey" button, (d) option to download the relevant checklist while they wait.

8. **No repeated referrals.** If a user has already submitted a lead for a professional type (e.g., mortgage broker at Stage 1), the Stage 2 CTA for the same professional changes to: "Your broker introduction is in progress. [Broker name] will contact you within 24 hours. Need to update your request?" This prevents duplicate leads.

9. **Professional matching logic (v2).** v1 sends leads manually via email to a panel of professionals. v2 introduces algorithmic matching based on: location, language, specialisation, availability, and user preference. ## CTA copy principles (applied across all touchpoints)

| Principle | Application |
|-----------|-------------|
| Names the specific need | "Borrowing assessment" not "talk to a broker" |
| User hires, does not defer | "Find a broker who can assess your situation" not "let a broker handle everything" |
| No urgency without reason | No countdown timers, no "limited spots" unless genuinely true |
| Transparency on cost | Where possible, note "free assessment" or "fixed fee" |
| Contextual relevance | Every CTA references what the user just learned on that page |
| Progression awareness | CTA acknowledges where the user is in the journey |
