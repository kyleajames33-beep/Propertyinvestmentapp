📸 App Store Screenshot Guide

High-quality screenshots are crucial for app store listings. This guide will help you capture the perfect screenshots for both iOS App Store and Google Play Store submissions.

📋 Screenshot Requirements

📱 iOS App Store

  • iPhone: 1290 × 2796 px (iPhone 15 Pro Max)
  • Alt: 1284 × 2778 px (iPhone 14 Pro Max)
  • Alt: 1242 × 2688 px (iPhone 11 Pro Max)
  • iPad: 2048 × 2732 px (12.9" iPad Pro)
  • Minimum: 3-10 screenshots
  • Format: PNG or JPEG

🤖 Google Play Store

  • Phone: 1080 × 1920 px minimum
  • Tablet: 1920 × 1080 px (landscape)
  • Minimum: 2-8 screenshots
  • Maximum: 8 screenshots
  • Format: PNG or JPEG
  • File size: Max 8MB each

⚠️ Important: Both stores recommend showing actual app functionality. Avoid heavily stylized mockups that misrepresent the actual user experience.

📷 What to Screenshot

Recommended Screenshots (in order):

  1. Stage 1: Financial Calculator - Empty State
    • Shows the main interface users will see first
    • Clean, professional input form
    • Demonstrates ease of use
  2. Stage 1: Financial Calculator - Filled with Sample Data
    • Use realistic Australian property values ($800,000 property)
    • Shows calculated results (borrowing capacity, LMI, etc.)
    • Displays the summary card with all metrics
  3. Stage 2: Suburb Comparison - Cards View
    • Add 3-4 suburbs with varied scores (high, medium, low)
    • Shows colorful score badges
    • Demonstrates comparison functionality
  4. Stage 2: Suburb Comparison - Table View
    • Toggle to table view
    • Shows compact data comparison
    • Demonstrates alternative viewing option
  5. Stage 2: Add Suburb Form
    • Shows data entry interface
    • Demonstrates input fields and validation
    • Optional: Include helper text

💡 Pro Tip: Add text overlays in a separate image editor (Canva, Photoshop) to highlight key features like "Calculate Your Borrowing Capacity" or "Compare 20+ Suburbs". This is optional but can improve conversion.

🛠️ How to Capture Screenshots

Method 1: Real Device (Recommended)

  1. Open the app on your iPhone or Android device
  2. Navigate to the desired screen
  3. Take a screenshot:
    • iPhone (Face ID): Press Side Button + Volume Up
    • iPhone (Home Button): Press Home + Power
    • Android: Press Power + Volume Down
  4. Screenshots are automatically saved to Photos/Gallery
  5. Transfer to computer via AirDrop, Google Photos, or USB

Method 2: Browser DevTools (Desktop)

  1. Open index.html in Chrome or Edge
  2. Press F12 to open DevTools
  3. Click the Device Toggle icon (or press Ctrl+Shift+M)
  4. Select device from dropdown:
    • iPhone 14 Pro Max (1290 × 2796)
    • Pixel 7 (1080 × 2400)
  5. Navigate to the desired screen in the app
  6. Right-click on the device frame → Capture screenshot
  7. Or use browser screenshot tool: Ctrl+Shift+P → type "screenshot" → select "Capture full size screenshot"

💡 DevTools Tip: If the screenshot resolution doesn't match app store requirements, you can set custom dimensions in DevTools. Click "Edit" → "Add custom device" → enter exact pixel dimensions required.

Method 3: Online Tools

📱 MockUPhone

Upload screenshots and add device frames. Free for basic use.

🖼️ Screely

Add browser chrome and backgrounds to screenshots. Good for desktop views.

🎨 Canva

Create custom app store graphics with text overlays and branding.

📲 AppMockUp

Professional device mockup generator with various templates.

✏️ Editing Screenshots (Optional)

While app stores prefer unmodified screenshots, you can enhance them:

Allowed Enhancements:

  • Add text overlays highlighting key features
  • Add device frames (iPhone/Android mockups)
  • Add subtle backgrounds (gradients, colors)
  • Crop/resize to meet exact store requirements
  • Add app name/logo at the top

Not Allowed:

  • Showing features that don't exist in the app
  • Heavily photoshopping/altering the UI
  • Using competitor app screenshots
  • Misleading functionality claims

💡 Recommendation: For v1.1 release, use clean, unedited screenshots. You can always update them later with enhanced versions.

💡 Tip: Number your screenshots (01, 02, 03...) so they upload in the correct order to app stores.

✅ Pre-Submission Checklist

  • All screenshots are correct dimensions for target store
  • Screenshots show real, working app functionality
  • At least 3 screenshots for iOS, 2 for Android
  • Screenshots are in PNG or JPEG format
  • File sizes are under 8MB
  • Screenshots are numbered in display order
  • No personal/sensitive data visible in screenshots
  • UI looks polished (no console errors visible, etc.)
  • Screenshots demonstrate key value propositions
  • Both Stage 1 and Stage 2 are represented
